# SignIn - 自動簽到模組

一個功能豐富的 Minecraft Fabric 自動簽到模組，支援多帳號自動切換、自動進入伺服器、自動存箱子等功能。

## 最新版本

最新版本：**2.0.0**

支持的 Minecraft 版本：**1.20.1**

## 功能特點

- ✅ 多帳號自動切換與管理
- ✅ 自動進入指定伺服器
- ✅ 自動執行簽到指令
- ✅ 自動將物品存入箱子
- ✅ 自動執行回家命令
- ✅ 任務完成後自動退出伺服器
- ✅ 可視化配置界面（支援 Mod Menu）
- ✅ 實時配置保存
- ✅ 操作日誌記錄與安全審計
- ✅ 自訂動作序列（支援移動、拿取、存放、按鍵等）
- ✅ 基於 Baritone 的自動導航移動

## 先決條件

- Minecraft 1.20.1
- Fabric Loader >= 0.15.8
- Fabric API
- Java 17 或更高版本
- Baritone API - 用於自動導航移動功能
- (可選) Mod Menu - 用於在遊戲中開啟配置界面

## 安裝

1. 確保已安裝 Fabric Loader 和 Fabric API
2. 下載並安裝 [Baritone API](https://github.com/cabaletta/baritone)（用於自動導航移動功能）
3. 下載最新版本的 `.jar` 文件
4. 將 `.jar` 文件放入你的 Minecraft 模組資料夾
5. 啟動遊戲

## 使用說明

### 開啟配置界面

你可以通過以下兩種方式打開配置界面：

- **方式一（推薦）**：在遊戲中輸入指令 `/dailyrewards config`
- **方式二**：安裝了 Mod Menu 模組後，在遊戲主選單的「模組」列表內找到 SignIn，點擊「配置」按鈕

### 主要配置區塊

#### 基礎參數 (General)

- 自動退出伺服器：任務完成後是否自動中斷連線
- 自動放置物品：是否自動將快捷欄物品存入最近的箱子
- 自動執行回家命令：任務結束後是否自動執行 `/home` 等指令
- 自動進入伺服器：啟動任務時是否自動連接到設定的伺服器
- 開箱物品欄位：自動存物時玩家手持的快捷欄索引 (0-8)

#### 指令設置 (Commands)

- 可在此修改登入、簽到、回家及退出指令，支持實時錯誤檢測

#### 帳號與伺服器管理 (Accounts & Servers)

- 支持通過 UI 列表新增、刪除或排序多個帳號與伺服器地址
- 直接拖動數字設置起始執行的帳號與伺服器

#### 自訂序列 (Custom Actions)

- 啟用自訂序列模式後可以完全自訂任務流程
- 支援多種動作類型：執行指令、移動至座標、自動拿取/存放物品、模擬按鍵
- 可自由排列動作順序，建立個人化的自動化流程

## 構建

如果你想從源代碼構建：

```bash
./gradlew build
```

構建完成後，`.jar` 文件將位於 `build/libs/` 資料夾中。

## 開發

克隆存儲庫：

```bash
git clone <repository-url>
cd signin-template-1.20.1
```

運行開發環境：

```bash
./gradlew genSources
./gradlew runClient
```

## 常見問題

**Q: 為什麼我的修改沒有生效？**

- A: 由於系統採用實時保存機制，若修改後立即強制關閉遊戲可能導致緩存未寫入。通常修改後看到右上角提示即代表已生效。

**Q: 如何快速切換帳號？**

- A: 在「帳號管理」區塊中，點擊左右箭頭或直接輸入數字。

**Q: 為什麼移動功能無法使用？**

- A: 移動功能需要安裝 Baritone API 模組才能使用。請確保已正確安裝 Baritone。
