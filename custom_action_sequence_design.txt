【自定義動作序列 (腳本化系統) 設計文檔】

此功能旨在為玩家提供高度自定義的自動化流程，允許玩家按順序排列一組動作，並在登入伺服器後自動執行。

一、 五大核心任務類型 (格式規範)
---------------------------------------
1. 加指令 (Command)
   - 格式: action:cmd:/你的指令
   - 說明: 支援所有帶有 / 的指令。
   - 實現: 使用 client.getNetworkHandler().sendChatCommand() 執行。

2. 自動移動 (Move)
   - 格式: action:move:x,y,z
   - 說明: 對接 Baritone 插件進行自動導航。
   - 實現: 發送 #goto x y z 指令，並監測玩家座標，直到抵達或卡住後繼續下一步。

3. 自動拿取 (Loot Chest)
   - 格式: action:loot
   - 說明: 尋找最近的箱子/桶子/陷阱箱。
   - 實現: 打開介面，遍歷箱子格子，使用 QUICK_MOVE (Shift+點擊) 將所有物品移入玩家物品欄 (Inventory)。

4. 自動存放 (Deposit Items)
   - 格式: action:deposit
   - 說明: 將玩家快捷列 (第 1-9 格) 的物品存入最近的箱子。
   - 實現: 掃描最近容器，將快捷欄內容快速移入箱中。

5. 模擬按鍵 (Press Key)
   - 格式: action:key:按鍵代碼
   - 說明: 支持常用按鍵如 f1-f12, q, e, 1-9 等。
   - 實現: 模擬物理按鍵按下與放開。

二、 GUI 界面設計 (配置分頁)
---------------------------------------
1. 主開關: 「啟用自定義序列模式」(開啟後取代原本的簽到/回家/存物流程)。
2. 快速添加按鈕區:
   - [➕ 加指令] [➕ 加移動] [➕ 加拿取] [➕ 加存放] [➕ 加按鍵]
3. 腳本序列列表:
   - 顯示所有已添加的動作。
   - 支持單行文字修改參數。
   - 支持 [⬆️ 上移] [⬇️ 下移] 排序動作。
   - 支持 [❌ 刪除] 動作。

三、 完整自動化循環流程
---------------------------------------
1. [啟動] -> [切換帳號] -> [進入伺服器] -> [登入成功]
2. [檢測模式]:
   - 若自定義模式開啟 -> 按列表順序 1, 2, 3... 執行動作序列。
   - 每個動作執行完畢後進入下一個。
3. [結束序列] -> [執行 autoLeave (若開啟)] -> [切換至下一個帳號] -> [循環開始]

四、 異常處理機制
---------------------------------------
1. 滿倉處理: 自動拿取時若玩家背包已滿，自動關閉箱子並執行下一個動作。
2. 找不到箱子: 若執行 loot/deposit 時周圍無容器，發送提示並自動跳過。
3. Baritone 缺失: 若執行 move 時未安裝 Baritone，發送警告並跳過。
